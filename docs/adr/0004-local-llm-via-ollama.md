# Local LLM (Ollama) for Claim Classification

Sentence classification is performed by a local model served via Ollama (e.g. Llama 3.1 8B, Qwen 2.5 7B+), not a cloud LLM. The reason a future reader might assume the opposite — "swap this out for Claude Sonnet, it's cheap and much better at classification" — is that the input is an **unpublished academic Paper**. Sending unpublished drafts to a third-party API risks preprint leakage, training-data contamination, and conflicts with several journals' confidentiality policies before submission. Local inference removes that risk entirely and also makes the tool offline-capable with zero per-call cost — useful because the **Author** runs the audit iteratively as the draft evolves. Trade-off accepted: classification quality is bounded by what runs on the Author's hardware, prompt-caching savings (Claude/OpenAI) don't apply, and the classifier prompt must be tuned against the local model's quirks. The classifier interface is kept pluggable so a cloud option can be added without architectural changes.
